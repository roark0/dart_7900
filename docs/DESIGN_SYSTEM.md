# 血液分析仪设计系统

## 文档信息

- 项目: 医疗设备 UI 设计系统
- 版本: 1.1.0
- 更新日期: 2026-03-24
- 目标设备: 800x600 触控屏
- 适用范围: 当前 Flutter 项目与后续同类仪器界面

## 1. 目标与原则

本设计系统用于统一医疗设备触控界面的视觉与交互表达，
避免页面各自实现导致风格漂移。

核心原则:

1. 可读性优先: 数据与状态信息必须一眼可读。
2. 一致性优先: 颜色、字号、控件状态统一。
3. 可复用优先: 新页面优先复用 token 和基础组件。
4. 可迁移优先: 规范尽量平台无关，不绑定单一前端框架。

## 2. 设计令牌

### 2.1 颜色令牌

主色:

- `navPrimary`: `#1A4F7A`
- `navActive`: `#3A9BD5`
- `tableHeader`: `#3A8FBF`
- `tableRowLight`: `#D0E8F5`
- `sidebarButton`: `#2D6FA0`
- `statusBar`: `#0D3A5C`

中性色:

- `background`: `#E8F2F8`
- `surface`: `#FFFFFF`
- `surfaceMuted`: `#F0F4F7`
- `border`: `#C0D8E8`
- `input`: `#D8D8D8`
- `foreground`: `#1E293B`

状态色:

- `disabledForeground`: `#888888`
- `success`: `#49BB3E`
- `error`: 建议使用系统错误色并保证与背景对比。

颜色使用规则:

1. 顶部导航与底部状态栏使用深色底。
2. 表头与激活态用高饱和蓝，避免与普通信息混淆。
3. 数据区优先白底或浅蓝底，文字保持深色。
4. 禁用态必须与可用态有显著差异。

### 2.2 字体令牌

基础字号层级:

- `navLabelSize`: `13`
- `tableHeaderSize`: `12`
- `dataValueSize`: `11`
- `statusSize`: `11`
- `sectionTitleSize`: `14`
- `buttonLabelSize`: `13`
- `fieldLabelSize`: `13`
- `inputValueSize`: `12`

字重建议:

- 导航标签: `w700`
- 表头与区块标题: `w600`
- 普通数据与状态: `w400`

字体策略:

1. 当前项目默认 `Arial`。
2. 若引入新字体，需一次性验证所有页面宽度占用。
3. 医疗设备场景不建议使用装饰性字体。

### 2.3 尺寸令牌

- `topBarHeight`: `74`
- `bottomBarHeight`: `34`
- `sideButtonHeight`: `48`
- `formFieldHeight`: `32`
- `tableHeaderHeight`: `40`
- `radius`: `4`
- `buttonRadius`: `10`

间距令牌:

- `space2`, `space4`, `space6`, `space8`, `space12`

## 3. 布局规范

### 3.1 屏幕结构

建议三段式结构:

1. 顶部导航区
2. 中部内容区
3. 底部状态区

内容区可按页面需求再拆分为:

- 左侧功能菜单
- 右侧主工作区

### 3.2 栅格与间距

- 主内容区统一内边距建议 `8` 或 `12`。
- 表格行高建议 `28`。
- 表头高度建议使用 token 值 `40`。
- 相邻操作按钮间距建议 `8`。

### 3.3 触控可用性

- 交互控件建议最小触控区不低于 `44x44`。
- 关键按钮需提供按下态和禁用态。
- 禁止把关键操作仅放在颜色变化上表达。

## 4. 组件规范

### 4.1 导航类组件

- 顶部主导航: 模块切换入口。
- 左侧子菜单: 页面内功能切换入口。
- 底部状态栏: 操作员、连接状态、时间。

### 4.2 数据展示组件

- 通用数据表格: 支持表头、斑马行、选中行。
- 三联参数表: 适用于 CBC/QC 类并列结果展示。
- 图表面板: 统一黑底与信息栏布局。

### 4.3 输入与操作组件

- 输入框: 固定高度，边框与背景使用 token。
- 按钮: 区分普通、主要、禁用三种状态。
- 分页条: 与表格联动，状态可见。

## 5. 页面模式

当前项目核心页面模式:

- `Analysis`: 数据总览 + 图表 + 标记
- `L-J QC`: 设置、分析、图形、列表
- `List Review`: 转置列表 + 筛选 + 分页
- `Maintenance`: 左侧菜单 + 内容工作区

页面新增规则:

1. 先复用现有结构组件，再补页面特有组件。
2. 新组件必须对齐已有颜色与字体令牌。
3. 新页面默认按 `800x600` 验证截图。

## 6. 当前项目落地

Flutter 实现入口:

- 令牌与主题: `lib/src/style/design_system.dart`
- 对外导出: `lib/src/style/palette.dart`
- 应用主题接入: `lib/src/app.dart`
- 外壳组件: `lib/src/widgets/instrument_scaffold.dart`
- 通用组件: `lib/src/widgets/common.dart`

约束说明:

1. 旧页面可继续使用 `UiPalette` 兼容别名。
2. 新代码优先使用 `UiTypography` 与 `UiMetrics`。
3. 如需改全局风格，先改 token，不直接散改页面常量。

## 7. 开发与验收

开发检查:

1. `flutter analyze`
2. 关键页面截图对比 `images/LQ_QC`
3. 检查按钮可点区域与文字可读性

文档检查:

1. `markdownlint docs/DESIGN_SYSTEM.md`

验收基线:

- 布局不溢出
- 字体层级清晰
- 颜色语义一致
- 交互状态完整

## 8. 变更流程

1. 先更新设计令牌。
2. 再更新组件实现。
3. 最后更新本设计文档与截图基线。

推荐在 PR 描述中附上:

- 变更页面
- 受影响 token
- 对比截图
- 风险说明
